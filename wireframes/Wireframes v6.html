flow · add wizard

One wizard. The input decides the path.

Word and sentence creation used to be two separate flows reached from two buttons. They're now one entry. Type or paste; the wizard detects whether you gave it a word or a sentence and branches. log · v5 → v6

What changed.

v5 quieted the streak. v6 is about consolidation — and filling in the rest of the app.

Merged add wizard One type-detecting entry replaces the separate add-word and add-sentence flows. Shared shell + image + review + save; branch only differs by 2 steps.
Two add entry points "+ Add a word" and "Paste a sentence" buttons. Now a single "+ Add".
Full review loop documented Intro → audio-only front → reveal & rate → brief → session done (cobalt).
Kind detector Live word/sentence detection under the input field; user-overridable.
Striped image placeholders Image candidates render as labelled striped slots in the handoff.
Carried from v5 Streak corner chip, period-scoped detail, surface-driven text rule, 4-stop heatmap ramp — all unchanged.

Open questions

  • Detector edge cases · a two-word lemma ("se calhar", "a gente") reads as a sentence by the token rule. Do we keep a short allow-list of multi-word lemmas, or let the user flip the chip?
  • Switching path mid-entry · if a draft is already generating on the word path and the user adds words to make it a sentence, do we discard the word draft or keep it parked? Leaning discard with an undo toast.
  • Sentence card register/tags · hidden by default on A.7·s. Is a single source tag enough, or do we surface register on sentence cards too?
  • "Write my own" sentence · on A.6 this drops i+1 guarantees. Warn, or trust the founder?
  • Share action on detail · still TBD what it produces. Hero card image is the obvious answer.

Removed since v5

  • Separate add-word flow · 8 frames, its own entry button
  • Separate add-sentence flow · 5 frames, near-duplicate shell
  • Two-button add affordance · collapsed to one "+ Add"
