# Runbooks

How-to guides for operational tasks.

Add a runbook when you find yourself doing something more than once
that requires non-obvious steps.

## Suggested runbooks (write as needed)

- `regenerate-frequency-list.md` — when corpus updates or schema changes
- `add-a-new-tts-provider.md` — recipe for adding a provider behind the boundary
- `bump-prompt-version.md` — when changing prompts substantively
- `migrate-schema.md` — recipe for Drizzle schema changes
- `validate-prompt-quality.md` — eval workflow before bumping prompts
- `seed-fresh-database.md` — getting a new dev DB to a working state
