# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Streak Detail + Design Fidelity.
> Generated by gsd-ui-researcher from codebase scan.
>
> **Important:** The v6 UI shell for this phase was shipped in Phase 1.1. This spec
> transcribes the already-implemented design contract. Phase 3's job is to replace
> the static placeholder data with real aggregation queries and apply the OLED +
> surface-text audit. No new visual surfaces are introduced.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage                                                |
| ----- | ----- | ---------------------------------------------------- |
| xs    | 4px   | Icon-to-text gap (hero line, chips), ramp swatch gap |
| sm    | 8px   | Between stat tiles, between sections, legend gap     |
| md    | 16px  | Card padded internal padding                         |
| lg    | 24px  | Section title bottom margin, nav padding-bottom      |
| xl    | 32px  | Scroll container bottom padding                      |
| 2xl   | 48px  | —                                                    |
| 3xl   | 64px  | —                                                    |

Exceptions (all captured from `app/streak/index.tsx` StyleSheet — do not normalize):

- **20px**: Screen horizontal padding (`paddingHorizontal: 20`). Not on the 4-grid; matches the v6 handoff phone frame width. Do not change to 16 or 24.
- **2–3px**: Heatmap cell gap (year: 2px, month: 3px) and reviews-chart bar gap (3px). Sub-4 values are intentional data-viz density decisions matching `wireframes/Wireframes v6.html` `.heatmap-year` and `.reviews-chart` specs. Do not normalize to 4px.
- **6px**: Period-toggle inner padding (3px container + segment pairing), best-row legend gap. Accepted micro-spacing for compound containers.
- **10px**: Best-row horizontal gap, hero gap to label column.
- **12px**: Hero horizontal padding, section margin-bottom between cards, stat-tile border-radius, card padded internal gap.
- **14px**: Best-row horizontal padding.

Source: `app/streak/index.tsx` StyleSheet + `wireframes/Wireframes v6.html`

## Surface Text Rule (DSGN-02)

The "text follows surface, not brand" rule is the central design constraint for Phase 3's audit pass:

| Surface         | Primary text | Muted text               | Faint text               | Brand emphasis                  |
| --------------- | ------------ | ------------------------ | ------------------------ | ------------------------------- |
| light (#FFF)    | `#000000`    | `rgba(0,0,0,0.60)`       | `rgba(0,0,0,0.42)`       | `#1F3494`                       |
| gold (#E8B838)  | `#1F3494`    | `rgba(31,52,148,0.65)`   | `rgba(31,52,148,0.45)`   | `#1F3494`                       |
| color (#2E5BC8) | `#FFFFFF`    | `rgba(255,255,255,0.75)` | `rgba(255,255,255,0.55)` | `#FFFFFF` (no cobalt-on-cobalt) |
| oled (#000000)  | `#FFFFFF`    | `rgba(255,255,255,0.70)` | `rgba(255,255,255,0.50)` | `#5A8FD4`                       |

Audit rule (DSGN-02): Any component that hardcodes `color: '#1F3494'` (cobalt) for body copy text on a light surface is a violation. Cobalt is permitted only for `tone="brand"` via the `textForSurface()` function on the `Num`, `Action`, and specific `Body tone="brand"` call sites listed above.

Source: `theme.ts textColors` + `Type.tsx useTextColor`

## Screen Layout Contract — Streak Detail

### Navigation bar

- Layout: `flexDirection: row`, `justifyContent: space-between`
- Left: `←` back arrow — `<Body tone="brand" weight="medium" size={14}>`
- Center: "Your streak" — `<Body weight="semibold" size={14}>`
- Right: `↗` share icon — `<Body tone="brand" weight="medium" size={14}>` (present, share action deferred — not in STRK-03..06)
- Border: `0.5px solid rgba(0,0,0,0.08)` bottom rule

### Compact hero

- Row: flame icon (Ionicons `flame`, 30px, `colors.accent`) + column (streak number + mono label line)
- Streak number: `<Num size={44}>` — Instrument Serif 44px, tone=brand
- Label line: `<Mono tone="muted" size={10}>` separated by `<Mono tone="faint">·</Mono>` separators
- Label content: `"{N} day streak · longest {M} · retention {R}%"`

### Period toggle

- Container: `backgroundColor: paperSoft`, `borderRadius: 9`, `padding: 3`
- Three segments: month / year / lifetime (exactly this wording, lowercase)
- Default active: `year`
- Active segment: `backgroundColor: paper`, `shadowOpacity: 0.06` elevation
- Active label: `<Body weight="medium" size={11} tone="primary">`
- Inactive label: `<Body weight="medium" size={11} tone="muted">`
- Text transform: capitalize on render (each label renders as "Month" / "Year" / "Lifetime")

### 2×2 Stat grid (period-scoped)

Uses `<StatTile>` component: `brandTint` background, `borderRadius: 12`, `padding: 12`.

| Period   | Tile 1                      | Tile 2                     | Tile 3                       | Tile 4                           |
| -------- | --------------------------- | -------------------------- | ---------------------------- | -------------------------------- |
| month    | longest streak (this month) | retention % (past 30 days) | total reviews (past 30 days) | days active (of N)               |
| year     | longest streak (date range) | retention % (past year)    | total reviews (past year)    | days active (past year)          |
| lifetime | longest streak (all time)   | retention % (all time)     | total reviews (all time)     | days active (since first review) |

- Value: `<Num size={28}>` — Instrument Serif 28px, tone=brand (cobalt)
- Label: `<Mono size={9}>` — Geist Mono 9px, uppercase, 0.10em letter spacing, tone=muted
- Sub-line: `<Body size={10} tone="faint">` — faint gray context text
- Grid: `flexWrap: wrap`, `gap: 8`, each tile `width: "48.5%"`

### Activity section (period-scoped)

**Year (default):** `<YearHeatmap>`

- Grid: 53 columns × 7 rows, `gap: 2px`, `borderRadius: 1.5px` per cell
- Today cell: `backgroundColor: transparent`, `borderWidth: 1`, `borderColor: colors.brand`
- Section header: "Activity" (left) + ramp legend (right: "less ▢▢▢▢ more")
- Month axis: sparse string labels, `size={8.5}` Mono faint, spaced evenly below grid

**Month:** `<MonthHeatmap>`

- Grid: 7-column calendar, `gap: 3px`, day `borderRadius: 3`, `aspectRatio: 1`
- DOW header: `['M','T','W','T','F','S','S']`, `size={8.5}` Mono faint
- Leading empty slots: `backgroundColor: transparent`
- Today cell: `backgroundColor: transparent`, `borderWidth: 1.5`, `borderColor: colors.brand`
- Section header: month name only (e.g. "May")

**Lifetime:** `LifetimeBars` — monthly bar chart

- Bars: `gap: 4`, `height: 80`, `backgroundColor: brandFill`, `opacity: 0.85`, radius 3px top
- Current bar (last): `backgroundColor: brand`, `opacity: 1`
- Section header: "Activity" (left) + "by month" (right, Mono muted)
- X-axis: year tick labels, brand color for last label

### Reviews bar chart (`ReviewsChart`)

- Container: `<Card padded radius={12}>`
- Section header: "Reviews" (Body semibold 13) + period meta (Mono muted 10)
  - month: "this month" | year: "past 12 months" | lifetime: "all time"
- Bars: `height: 50`, `gap: 3`, `backgroundColor: brandFill`, `opacity: 0.9`, radius 2px top
- Current bar (last): `backgroundColor: brand`, `opacity: 1`
- Axis labels: `size={8}` Mono, faint for all except last (brand color)

### Rating distribution (`<RatingDistribution>`)

- Container: `<Card padded radius={12}>`
- Section header: "Rating distribution" (Body semibold 13) + "{N} reviews" (Mono muted 10)
- Bar: `height: 14`, `borderRadius: 4`, flex-row segments sized by percentage
- Segment order: again / hard / good / easy (left to right)
- Legend: 4-column flex, per-rating: swatch (8×8px, borderRadius 2) + uppercase Mono name (9px) + bold percentage (Mono 13px, tone=primary)
- Empty state (0 reviews): single `paperSoft2` bar + all legend shows "0%"

### Personal bests (`<Card radius={12}>`)

- Each `BestRow`: `paddingVertical: 10`, `paddingHorizontal: 14`, `gap: 10`
- Current row: `backgroundColor: brandTint`
- Rank column: `<Mono tone="faint" size={10}>`, width 18
- Streak value: `<Num size={20}>` + `<Body tone="muted" size={11}> days</Body>`
- "current" badge (if applicable): `<Mono tone="brand" size={9}> current</Mono>` below streak value
- Date range: `<Mono tone="muted" size={10}>`, margin-left auto
- Last row: no bottom border

### OLED review screen (DSGN-01)

- Trigger: `useNightSurface()` — returns `'oled'` when `colorScheme === 'dark'` AND `hour >= 19 || hour < 6`
- Applied to: review screen (`app/review/index.tsx`) background + elevated surfaces only
- Screen bg: `colors.oled` (`#000000`)
- Elevated surfaces (rating buttons, card reveal): `colors.oledElevated` (`#0A1422`)
- Text primitives: all surface-aware components automatically adapt via `ThemeContext`
- Streak detail: NOT affected — light surface always

Source: `app/review/index.tsx` + `src/lib/useNightSurface.ts`

## Data States Contract

Phase 3 wires real aggregation. These states must render correctly with real data:

### New user / empty data state (0 reviews, 0-streak)

| Element             | Empty rendering                                                            |
| ------------------- | -------------------------------------------------------------------------- |
| Hero streak number  | `0` (Num size={44}, still brand cobalt — the number IS the data)           |
| Hero meta line      | "0 day streak · longest 0 · retention 0%"                                  |
| Stat tiles          | Show real zeros: "0" longest, "0%" retention, "0" reviews, "0" days active |
| Heatmap             | All cells heat-0 (`#EFF3FB`), today cell outlined (still renders)          |
| Reviews chart       | All bars at min-height (3–6px zero-height stub), still renders             |
| Rating distribution | Empty bar (paperSoft2 fill), all legend shows "0%"                         |
| Personal bests      | Card renders with one current-row entry: rank "→", "0 days", "today" date  |

Empty state behaviour:

- No separate empty screen, no illustration, no redirect — zeros display inline
- Stat tile sub-lines remain (e.g. "of 31" for days active of month)
- Personal bests always shows the current-row entry even with 0 days so the screen is never blank

### Loading + error state

The stats screen is non-critical: it has no `RefreshControl` / pull-to-refresh (`ScrollView` only). The policy is simple:

- **On load:** show zeros until the aggregation query resolves, then transition to real data
- **On query error:** keep zeros displayed; no error UI, no toast on this screen
- No skeleton loaders — zeros are the visual placeholder

## Interaction Contract

### Period toggle

- Tap any segment → `setPeriod(p)` → stat grid, activity chart, reviews chart, and rating distribution all re-render to the new period's data
- No animation on switch (instant data swap)
- Personal bests: NOT period-scoped — always shows all-time bests

### Heatmap cells

- No tap interaction defined (non-interactive, display only)

### Review screen OLED auto-switch

- No user gesture — automatic on enter (`useNightSurface()` reads color scheme + hour on mount)
- No manual override in Phase 3 scope

### Navigation

- Back (←): `router.back()`
- Share (↗): present, no action in Phase 3

## Audit Checklist (DSGN-02 text rule pass)

Phase 3 includes an app-wide audit. Violations to search for:

- Any hardcoded `color: '#1F3494'` or `color: colors.brand` on non-Num/non-Action/non-key-action text elements
- Any `<Text style={{ color: colors.brandFill }}>` used for body copy
- Cobalt-colored body text on `paperSoft`/`brandTint` backgrounds (should be `textColors.light.primary` = `#000000`)
- Text primitives (`Body`, `Mono`, `Num`, `Action`, `Display`) called without explicit `surface` prop inside surfaces that differ from the ThemeContext value — verify context provider wraps the correct scope
