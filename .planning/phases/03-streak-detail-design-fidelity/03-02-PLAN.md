`api.getStreakStats()`. The route fetches the user's cards, scopes reviews by those card ids
(security), computes the three period windows (D-07), and returns month/year/lifetime stats
plus all-time personal bests in a single payload so the screen's period toggle is instant.

screen with a single query. No schema change.

`getStreakStats()` in `apps/mobile/src/lib/api.ts`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile client → API (`GET /streak/stats`) | Untrusted client requests aggregated review data. The route must return ONLY the requesting user's data. |
| API → Postgres (reviews/cards) | `reviews` has no `user_id`; cross-user leak if reviews are read without scoping by the user's card ids. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-02 | Information Disclosure | `streakRoute.get('/stats')` reviews read | mitigate | Scope reviews to the user via `inArray(reviews.card_id, userCardIds)` where userCardIds derives from `cards` filtered by `V0_USER_ID`; return [] when the user has no cards. Mirrors home.ts T-02-01. Verified by grep that the route never calls `db.query.reviews.findMany` without the inArray scope. |
| T-03-03 | Tampering | npm/pip/cargo installs | mitigate | No new packages added in this plan (Hono, drizzle, db already vetted in Phase 1/2). No legitimacy checkpoint required. |
</threat_model>
