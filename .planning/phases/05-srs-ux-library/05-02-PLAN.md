DECISION — sentence-edit audio (RESEARCH Open Question 1): Option A (re-synthesize) is chosen, consistent with Phase 4 plan 4.2 precedent where Add-wizard sentence edits regenerate audio. Audio is content-addressed by SHA-256(sentence); a stale hash plays the old sentence, which breaks the audio-first review loop. The synthesize call is CONDITIONAL — only when `sentence_pt` is present in the patch AND differs from the stored value. A `source_tag`-only edit must NOT call Narakeet.
</objective>
