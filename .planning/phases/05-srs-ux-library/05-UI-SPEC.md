## Spacing Scale

All values sourced from live NativeWind class usage in `cards/index.tsx`, `add/index.tsx`, and `Surface.tsx`. The codebase uses a mix of 4pt-grid values — active tokens listed below.

| Token | Value | Usage                                                                           |
| ----- | ----- | ------------------------------------------------------------------------------- |
| xs    | 4px   | Icon gaps, gender-rail top/bottom inset                                         |
| sm    | 8px   | Compact chip gap, button inner padding                                          |
| md    | 16px  | Screen horizontal padding (`px-4`), card padded inset, left-rail content offset |
| lg    | 24px  | Section top padding (`pt-4` = 16 + safe area; section breaks)                   |
| xl    | 32px  | FlatList bottom padding, major section breaks                                   |
| 2xl   | 48px  | (reserved; available on scale)                                                  |
| 3xl   | 64px  | (reserved; available on scale)                                                  |

Exceptions:

- `44px` minimum tap target for all interactive elements (iOS HIG). Swipe row actions must span the full row height, minimum 44pt.
- NativeWind default passthroughs (not available for new design decisions in this phase):
  - `12px` (`p-3` / `gap-3`): Stock Tailwind value already shipped in StatTile row padding and filter chip vertical gap. Not available for new design decisions in this phase; use 8px or 16px for new elements.
  - `20px` (`px-5`): Stock Tailwind value already used at screen horizontal edge padding in `cards/index.tsx`. Not available for new design decisions in this phase; use 16px or 24px for new elements.

## Typography

Four principal sizes cover all text roles in this phase. All one-off size props in implementation must resolve to the nearest declared principal size.

| Role                  | Principal Size | Weight         | Font                                       | Line Height |
| --------------------- | -------------- | -------------- | ------------------------------------------ | ----------- |
| Display               | 30px           | regular (400)  | Instrument Serif / Instrument Serif Italic | ×1.05       |
| Num                   | 28px           | regular (400)  | Instrument Serif                           | ×0.95       |
| Body                  | 15px           | regular (400)  | Geist                                      | ×1.5        |
| Action / Label / Mono | 11px           | semibold (600) | Geist SemiBold / Geist Mono                | 16px fixed  |

**Two declared weights: regular (400) and semibold (600).** Medium (500) exists in `fonts.uiMedium` but is not a declared principal weight — implementations must use 400 or 600.

**Size collapse notes:**

- `<Heading>` (section heading variant, previously 22px): render at 30px using the Display role. Pass `size={30}` to the Display primitive or use the `<Heading>` alias — do not declare 22px as a separate size.
- `<Num>` stat display: 28px everywhere, including large standalone stat on the card detail screen. If a visual size difference is needed for a hero stat, pass a runtime `fontSize` prop to the primitive — this is not a declared spec size and must not be added to this table.
- `<Mono>` eyebrow labels: render at 11px (Action / Label / Mono role). Do not use 10px.
- `<Body>` sub-labels: use 15px with reduced `opacity` or muted `color` tone to visually de-emphasize — do not reduce size below 15px.
- `<Action>` (14px in `Type.tsx`): rounds to the Action / Label / Mono role at 11px for spec purposes. Implementation may render at 14px via the primitive's internal size but must not declare 14px as a new spec size.

## Color

Source: `theme.ts` `colors` object + `textColors` map. Rule: **text follows surface, not brand.**

Primary focal point: the filter chip row immediately below the "Cards" screen title — it is the first interactive element on the screen and the primary visual anchor for library navigation.

| Role                    | Value                                                           | Usage                                                              |
| ----------------------- | --------------------------------------------------------------- | ------------------------------------------------------------------ |
| Dominant surface (60%)  | `#FFFFFF` (`colors.paper`)                                      | Screen background, card surfaces, list rows                        |
| Secondary surface (30%) | `#F7F7F8` (`colors.paperSoft`) / `#EFF3FB` (`colors.brandTint`) | Default Chip background, StatTile background, inactive filter pill |
| Accent — cobalt (10%)   | `#1F3494` (`colors.brand`) / `#2E5BC8` (`colors.brandFill`)     | See reserved-for list below                                        |
| Destructive             | `#C84A40` (`colors.again`)                                      | Delete swipe action background, delete confirmation button         |

**Accent cobalt reserved for (explicit list — no other uses):**

1. Active filter chip: filled cobalt `#1F3494` background, white text
2. SRS stat numbers on card detail (via `<Num>` tone="brand")
3. "Clear filters" text button (filtered-to-zero empty state)
4. Masculine gender rail (`colors.brand` = `#1F3494`)
5. "New" SRS state badge text and tinted background (`rgba(31,52,148,0.13)`)

**Suspend swipe action is neutral/muted (non-destructive) — not accent.** Background: `rgba(0,0,0,0.08)`, icon/label: `rgba(0,0,0,0.60)`.

**Semantic SRS state badge colors** (canonicalized from existing `cards/index.tsx` pattern):

| SRS State  | Text color                      | Background              | Note                                                            |
| ---------- | ------------------------------- | ----------------------- | --------------------------------------------------------------- |
| new        | `#1F3494` (`colors.brand`)      | `rgba(31,52,148,0.13)`  |                                                                 |
| learning   | `#C99A1F` (`colors.accentDeep`) | `rgba(201,154,31,0.13)` | Canonicalized from shipped `#F0BF38` to the nearest theme token |
| relearning | `#C99A1F` (`colors.accentDeep`) | `rgba(201,154,31,0.13)` | Same as learning                                                |
| review     | `#22C55E`                       | `rgba(34,197,94,0.13)`  | Hardcoded green — no theme token; do not change in Phase 5      |
| suspended  | `rgba(0,0,0,0.40)`              | `rgba(0,0,0,0.06)`      | Muted gray — inactive/neutral                                   |

**Suspended badge coexistence rule:** When a card's `suspended_at` is not null (D-08), the "Suspended" badge _replaces_ the SRS state badge in the library row. A suspended card does not show both. On the card detail screen, the SRS State stat tile shows "Suspended" in place of the state value.

**Gender rail colors** (existing, do not change):

- Feminine: `#B43A6C` (`colors.genderFem`)
- Masculine: `#1F3494` (`colors.brand`)
- Common: `#C99A1F` (`colors.accentDeep`)

**Gold `#E8B838` (`colors.accent`) is NOT used in Phase 5.** Gold is reserved for audio/play elements in the review flow. The library and card detail screens are light-surface only.

**OLED theme:** Not triggered in Phase 5. Library and card detail are always light surface.

## Interaction Contract

### Library screen — filter chips

Source: CONTEXT.md D-02; mirrors existing chip pattern at `cards/index.tsx:40-57`.

- Chips are horizontally scrollable in a single row (`ScrollView horizontal showsHorizontalScrollIndicator={false}`)
- Four chip groups: **gender** (Masculine / Feminine / Common), **source tag** (WhatsApp / Instagram / Netflix / custom values), **register** (Formal / Informal / Slang / Neutral), **SRS state** (New / Learning / Review / Relearning / Suspended)
- All chips rendered with `<Chip>` component
- **Inactive chip**: `variant="default"` — `paperSoft` background, `rgba(0,0,0,0.08)` border, black text
- **Active chip**: override to cobalt fill — `backgroundColor: #1F3494`, `borderColor: #142468`, text white (`#FFFFFF`), font `fonts.uiSemiBold` (600)
- Multiple filters may be active simultaneously (AND logic, CONTEXT.md D-02)
- Horizontal padding matches screen edge: 20px left, 20px right, 8px gap between chips

### Library screen — default sort

Newest-first by `created_at` descending. No search bar in this phase (CONTEXT.md: "keep it simple").

### Library screen — row swipe

Mechanism: `react-native-gesture-handler` `Swipeable` (v2.28.0, already a project dependency). No existing swipe pattern in codebase — Phase 5 introduces it.

- Swipe **left** reveals two action buttons (CONTEXT.md D-03)
- **Suspend/Unsuspend action**: 80px wide, `backgroundColor: rgba(0,0,0,0.08)`, icon `pause-circle` (when unsuspended) / `play-circle` (when suspended) from Ionicons, muted gray label, `color: rgba(0,0,0,0.60)`
- **Delete action**: 80px wide, `backgroundColor: #C84A40`, icon `trash` from Ionicons, white label
- Row tap opens card detail screen (no swipe needed for tap)
- Swipe reveals actions without executing; delete requires tapping the Delete button (which then triggers the Alert confirmation)
- Suspend executes immediately on tap — no confirmation (reversible action)

**Swipe label copy (width constraint note):** The 80px panel width fits single-word labels comfortably. "Suspend" / "Unsuspend" / "Delete" are used on the swipe panel. The card detail screen uses noun-qualified versions: "Suspend card" / "Unsuspend card" / "Delete card" where width is unconstrained. Do not add "card" noun to the 80px swipe panel labels.

### Library row anatomy

Extends existing `CardRow` from `cards/index.tsx`:

- 3px gender rail (left edge, `colors.genderFem` / `colors.brand` / `colors.accentDeep`)
- Headword in `<Body weight="semibold">` at 15px — primary tone
- Gendered form / sub in `<Body>` at 15px — muted tone (use opacity/color, not a smaller size)
- **SRS badge**: when `suspended_at` is null, show SRS state badge with semantic colors. When `suspended_at` is set, show "Suspended" badge (muted gray) instead — do not show both.
- Right: no chevron (swipe affordance sufficient per iOS convention)

### Card detail screen

Route: `app/cards/[id].tsx` (Expo Router, CONTEXT.md canonical refs)

Layout (top to bottom, all on light surface):

1. **Image thumbnail** — 100% width, 180px height, `borderRadius: 0` at top (full-bleed), `resizeMode: cover`. If no image: `paperSoft` placeholder 180px with muted "No image" `<Body>` label centered.
2. **Word header section** — 20px horizontal padding, 16px top: headword as `<Display>` at 30px, gendered form as `<Body>` at 15px muted tone. 3px gender rail on left edge of this section.
3. **SRS stats grid** — 2 columns × 3 rows of `<StatTile>` components, 12px gap (NativeWind `gap-3` passthrough), 20px horizontal padding (NativeWind `px-5` passthrough), 20px top. Six stats: Stability, Difficulty, SRS State, Due Date, Total Reviews, Last Reviewed. (See formatting table below.)
4. **Editable fields section** — sentence (`sentence_pt`) as a `<TextInput>` styled to match `<Body>` at 15px; source tag chip row (same chip pattern as add wizard). Section header: `<Mono>EDIT</Mono>` eyebrow label.
5. **Actions** — two full-width buttons stacked, 20px horizontal margin: "Suspend card" / "Unsuspend card" as a muted bordered button (`borderColor: rgba(0,0,0,0.15)`, text `rgba(0,0,0,0.60)`); "Delete card" as a destructive text-only button in `colors.again` (`#C84A40`).
6. Navigation: standard `<Stack.Screen>` back arrow from Expo Router. No additional header buttons.

### SRS stat formatting (D-05, value display rules)

FSRS difficulty is stored as a float; the FSRS-4.5 spec defines difficulty on a scale of 1–10.

| Field                | Display format                                                                                |
| -------------------- | --------------------------------------------------------------------------------------------- |
| stability            | `Math.round(stability)` + " days" (e.g. "14 days")                                            |
| difficulty           | `Math.round(difficulty)` as integer 1–10, no unit (e.g. "6")                                  |
| state                | SRS state badge chip (see semantic color table); shows "Suspended" when `suspended_at` is set |
| due_at               | Past or today → "Today"; tomorrow → "Tomorrow"; else → locale short date (e.g. "Jan 3")       |
| reps (total reviews) | Integer, no unit (e.g. "23")                                                                  |
| last_reviewed_at     | null → "Never"; today → "Today"; yesterday → "Yesterday"; else → relative (e.g. "3 days ago") |

### Pull-to-refresh

The error state copy says "Pull to refresh." — the library FlatList must implement `<RefreshControl>` on the FlatList to support this. Refresh invalidates the `['cards', 'all']` TanStack Query key. This is the only new interaction added to the FlatList refresh pattern; no separate "Retry" button is needed.
