<threat_model>

## Trust Boundaries

| Boundary                              | Description                                          |
| ------------------------------------- | ---------------------------------------------------- |
| mobile clipboard → TextInput          | OS clipboard content injected into sentence field    |
| mobile → POST /cards edits.source_tag | User-supplied or chip-selected label persisted to DB |

## STRIDE Threat Register

| Threat ID | Category               | Component                                          | Disposition | Mitigation Plan                                                                                                                       |
| --------- | ---------------------- | -------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| T-04-07   | Tampering              | Clipboard content injected into sentence TextInput | accept      | Clipboard is read-only injection into a text field the user controls; they can clear it; no eval or SQL surface                       |
| T-04-08   | Information Disclosure | Clipboard read without explicit prompt             | mitigate    | Only reads on mount when kind=sentence; immediately visible to user in TextInput; no background read; user can see and clear the text |
| T-04-09   | Tampering              | source_tag text column — free-form input           | accept      | source_tag is stored as plain text, not used in queries or code paths with injection risk; max length not enforced (low-value column) |
| T-04-10   | Spoofing               | DB migration alters cards table schema             | mitigate    | Migration adds nullable column with no default — backwards compatible; existing rows get null source_tag safely                       |
| T-04-SC   | Tampering              | expo-clipboard npm package                         | mitigate    | Blocking human checkpoint (Task 0) verifies package legitimacy on npmjs.com + Expo docs before install                                |

</threat_model>
