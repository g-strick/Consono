## Tests

### 1. Cold Start Smoke Test

expected: Start Expo from scratch (e.g. `npx expo start -c`, open in simulator/Expo Go). App boots with no red-box error, custom fonts load, and Home renders the v6 layout (not a flash of system fonts that never resolves).
result: pass (after fix)
reported: "Unable to resolve module buffer from react-native-svg/src/utils/fetchData.ts — import { Buffer } from 'buffer' could not be found within the project"
severity: blocker
root_cause: "react-native-svg@15.11.2 resolves to its TS src entry under React Native; src/utils/fetchData.ts imports the `buffer` polyfill, which is not a dependency. The only use of react-native-svg was a static decorative waveform on the review front (plan 04). Metro red-boxes on boot, so the app never renders."
fix: "Replaced the Svg/Path waveform in app/review/index.tsx with a View-based bar row and removed the unused react-native-svg dependency from package.json. No polyfill / host pnpm install required — the broken module is no longer in the bundle graph. Commit d19458d. tsc clean."
