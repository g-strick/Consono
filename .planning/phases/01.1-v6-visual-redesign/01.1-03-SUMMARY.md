Pure detectKind() implementing the v6 detector contract, and a full rewrite of add/index.tsx
collapsing the two separate word/sentence wizards into one shared shell with live detection,
an overridable detector chip, and complete v6 token reskin across all 6 steps.

## What Was Built

**Task 1 — detectKind():**

- `src/lib/detectKind.ts`: pure function, no React, imports `CardKind` from `@/src/lib/api`
- Contract: empty → `'word'`; ≤1 whitespace-separated token + no terminal punctuation → `'word'`; else → `'sentence'`
- Terminal punctuation: `.` `!` `?` `…`; hyphen is not a token separator (`'bem-vindo'` → word)
- Co-located `detectKind.test.ts` using typed node assertions (13 cases covering all contract edges)

**Task 2 — Input shell + live detector:**

- `Mode` state removed; `kindOverride: CardKind | null` replaces it
- Derived `kind = kindOverride ?? detectKind(inputText)` recomputes on every keystroke
- `handleInputChange` clears override on field empty (detector contract: "clear override when field cleared")
- `handleKindOverride` flips kindOverride to the opposite of current detected kind
- Word path: Display 30px serif input field
- Sentence path: Card-wrapped `TextInput` with Instrument Serif 17px
- Detector chip: colored dot + label (`word · we'll handle gender` / `sentence · N words · no gender step`); tappable
- Word info banner: paperSoft flat card showing "building a word card" + Mono "4 steps"
- Sentence source chips: whatsapp/instagram/netflix/+ tag (display-only, Phase 4 deferral)
- Empty state: recent chips (tappable, populate field) + frequency brand chips
- Wiz-top: `Action` ← Cancel/Back + `Mono` step label; 3px progress bar (brand; gold loading)
- `stepLabel()`: word → 2/4 3/4 4/4; sentence → 2/3 3/3; based on derived kind

**Task 3 — All remaining wizard steps reskinned:**

- `LoadingStep`: two pipeline-row layouts (word: lemma/gender/stress/images/i+1; sentence: parsed/query/images/gloss); static placeholders per spec; gold progress bar
- `PickImageStep`: 2×2 image grid; 2px brand border + brand `✓` badge on selected; hint text; display-only ↻/↑/⌕ chips; gender chip in header for word kind
- `PickSentenceStep`: `i+1` brand Chip header; radio dot rows; selected row = brand border + brandTint bg; display-only ▶/↻/✎ chips
- `ReviewStep`: word → gender `Card` (3px rail) with serif headword, gender Chip, stress Mono, image, sentence, audio placeholder, register Chips + Save row; sentence → plain `Card` image-first + serif headline
- `SavedStep`: brandTint 72px circle with cobalt ✓; Display "In your queue."; card id + gender `Card` preview; `+ Another` / `▶ Review` CTA row
- `api.generate` / `api.approveCard` request shapes unchanged; `approveMutation` wiring preserved

## Deviations from Plan

**1. [Rule 3 - Blocking] @ts-nocheck rejected by eslint ban-ts-comment rule**

- Found during: Task 1 commit
- Issue: Wrote `detectKind.test.ts` with `@ts-nocheck` (no `@types/jest` installed). ESLint's `@typescript-eslint/ban-ts-comment` rule rejected the file at pre-commit.
- Fix: Rewrote the test using a typed `assertEqual<T>()` helper with plain node assertions instead of Jest globals. No framework needed; file compiles and runs under `node -r ts-node/register`.
- Impact: Test file is now typed and immediately runnable; Jest migration is a one-line change.

**2. [Auto] Tasks 2 and 3 combined into single commit**

- Found during: implementation — both tasks modify the same single file `add/index.tsx`
- Fix: Wrote the full v6 wizard (all steps) before first commit to avoid a partial-broken state in git. Combined into one `feat` commit with both task descriptions.
- Impact: None — tsc passes end-to-end; the single commit cleanly replaces the file.

**3. [Rule 1 - Bug] PipelineRow type needed explicit annotation**

- Found during: tsc run in Task 2
- Issue: TypeScript could not narrow the `chip`/`mono`/`bold` optional properties from array literal inferences.
- Fix: Added explicit `PipelineRow` interface type; annotated the two row arrays.
- Impact: Compile error eliminated; no behavior change.

## Known Stubs

- **LoadingStep pipeline rows** — all "✓ lemma / ✓ gender / ○ images" etc. are static visual placeholders. Real live pipeline status is Phase 4 ADD-01. The gender chip shows "feminine" hardcoded until the API returns live status.
- **Source chips** (whatsapp/instagram/netflix) — display-only, not persisted to backend. Phase 4 will wire real source tagging.
- **Action chips** (↻ more / ↑ upload / ⌕ refine on PickImageStep; ▶ play all / ↻ more / ✎ write own on PickSentenceStep) — display-only per plan; functionality deferred to later phases.
- **Audio button in ReviewStep** — placeholder `▶` button; actual audio playback driven by existing `audio_url` field wired in Phase 1.2 (review screen). No audio regression — the approve call is unchanged.

## Threat Flags

None — `detectKind` is a pure string classifier with no eval, no network, no injection sink. Source chips are display-only and send nothing to the server.
