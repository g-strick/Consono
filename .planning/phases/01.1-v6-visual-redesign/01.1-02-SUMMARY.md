## What Was Built

**Task 1 — StreakChip reskin:**

- Replaced hardcoded `#1F3494` fill with `colors.brandFill` (`#2E5BC8`)
- `at-risk`: `colors.brand` 1.5px border, cobalt text
- `inactive`: `colors.grayRule` 1px border, `rgba(0,0,0,0.60)` text
- `continued`: `colors.brandFill` background, white text, no border, 10px h-padding
- `Animated.timing` (280ms, ease-out, `useNativeDriver: false`) interpolates `backgroundColor` and text color (`fromTextColor → #FFFFFF`) via `Animated.Text`
- Count updates instantly (no number animation)
- `fonts.uiMedium` for count text; flame icon via `Ionicons` at 13px
- Existing `StreakState` type and `{ count, state }` props unchanged; `router.push('/streak')` tap preserved

**Task 2 — StatTile, Chip, Card:**

- `StatTile({ value, label, sub? })`: `colors.brandTint` bg, radius 12, padding 12; `Num` primitive (28px cobalt); `Mono` primitive (9px, 0.10em spacing) for label; optional `Body` (10px faint) for sub
- `Chip({ label, variant?, color? })`: pill radius 999; default variant (`paperSoft` + `rgba(0,0,0,0.08)` divider); brand variant (`rgba(31,52,148,0.08)` bg + cobalt text); `color` override derives tinted bg via `hexToRgba()` for gender chips
- `Card({ children, gender?, padded?, radius? })`: `colors.paper` bg, `0.5px rgba(0,0,0,0.08)` border, configurable radius (default 14); gender prop adds absolute 3px left rail (`genderFem`/`brand`/`accentDeep`) per `.gbar` wireframe rule

**Task 3 — RatingButtons:**

- 4-column `flexDirection: 'row'` layout with `gap: 6`
- Each `Pressable`: `borderRadius: 10`, `borderWidth: 0.5`, semantic `borderColor` and `backgroundColor` per `RATING_CONFIGS` lookup
- `again`: `colors.again` red border + label; `hard`: `colors.hard` gold border + `colors.accentDeep` label; `good`: `colors.good` border + label + `colors.brandTint` bg; `easy`: `colors.easy` border + `colors.brandSoft` label
- Label: `fonts.uiSemibold` 12px; interval: `fonts.monoMedium` 9.5px muted
- `Rating` type imported from `@/src/lib/api`; all semantic colors from `theme.ts`

## Deviations from Plan

**1. [Rule 1 - Limitation] Ionicons flame icon cannot animate color via Animated.Value**

- Found during: Task 1
- Issue: `Ionicons` `color` prop accepts a plain `string`, not `Animated.AnimatedInterpolation`. Wrapping with `Animated.Text` is only valid for native `Text` nodes.
- Fix: Flame icon color is set statically per state (`colors.brand` / `rgba(0,0,0,0.60)` / `#FFFFFF`) rather than interpolated. The text color IS animated via `Animated.Text`. The visual result matches the wireframe since the icon color snaps instantly on state change (the fill animation masks any abruptness). The spec says "text crossfades cobalt → white" — the count text still does this.
- Impact: Minor — icon color does not animate smoothly on the 280ms transition but the fill animation is the primary visual cue per the wireframe description.
