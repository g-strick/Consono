<objective>
Establish the database layer for Supabase Auth: a journal-registered Postgres trigger migration that
auto-creates a `public.users` profile row whenever Supabase Auth inserts into `auth.users`, and an
explicit, user-confirmed decision on the RLS roadmap deliverable (RESEARCH Open Question 4, in tension
with ADR 0007 rule 3) — applied as policies if the user chooses that option.

Purpose: The trigger keeps existing FK relationships intact when real users sign up (RESEARCH Pattern 8).
The RLS decision is a roadmap-listed Phase 6 deliverable ("Row-level security on all DB tables") that
RESEARCH shows is a silent no-op under the postgres/BYPASSRLS role — it must be resolved with the user,
not dropped or stubbed; if option-a is chosen, the policies are written and applied in this plan.
Output: A journal-registered custom SQL migration with the trigger, a recorded RLS decision, and
(conditionally) an applied RLS policy migration.
</objective>
